# 趋势雷达 Design Brief

## Product Positioning

趋势雷达是面向国内用户的信息情报产品，核心价值是把开源项目、技术趋势、新闻信号和可复用资源压缩成可阅读、可判断、可行动的每日信息流。界面应当像一个专业情报工作台，而不是博客、营销页或普通内容站。

## Visual Principles

- 克制、清晰、专业，避免炫技式装饰。
- 信息密度要高，但层级必须清楚。
- 首页先回答“今天有什么值得看”，详情页再展开判断。
- 卡片只用于具体信息单元，不把大段页面包进卡片。
- 表格和榜单要紧凑，文字过长时截断或进入详情页，不撑宽布局。
- 动效只用于状态变化、hover 和进入反馈，不能干扰阅读。

## Brand Tone

- 中文名称：趋势雷达
- 气质关键词：专业、冷静、可信、长期跟踪、研究判断
- 避免：过度科技感、深色大屏、夸张渐变、营销式大标题、信息搬运感

## Layout System

- 页面最大内容宽度建议 1180px。
- 主要页面采用上下结构：顶部导航、摘要区、核心榜单、分主题模块。
- 榜单优先双列布局，移动端改为单列。
- 项目详情页采用主内容 + 右侧指标栏，移动端右侧指标下移。
- 返回按钮、面包屑或上级入口必须明显，避免用户进入详情后迷路。

## Color System

- 背景：接近白色的暖灰或浅绿灰。
- 主文字：近黑，不使用纯黑大面积压迫。
- 辅助文字：低饱和灰绿。
- 强调色：低饱和深绿，可少量搭配金色用于星标。
- 避免单一绿色铺满页面；不要使用紫蓝渐变、霓虹色、重阴影。

## Typography

- 中文正文优先使用系统字体：PingFang SC / Microsoft YaHei / system-ui。
- 标题可以粗，但不要过大导致压迫。
- 榜单卡片标题只放项目名；作者、语言、星标等放次级信息。
- 不使用负字距；不根据视口宽度无限缩放字体。

## Components

- Ranking Board：展示排名、项目名、作者、星标数。卡片无明显轮廓，使用轻微分隔和 hover。
- Project Detail：顶部展示返回入口、项目名、作者、标签、摘要；正文只保留对用户有判断价值的模块。
- Metrics Panel：右侧只展示关键指标，不重复正文。
- Resource Links：GitHub 官方链接、官方文档、NAS 镜像链接分组展示。
- Source Modules：GitHub、新闻、X/社区趋势等模块视觉一致，但数据来源清晰标注。

## Content Rules

- 不做 README 翻译站。每条内容必须回答：解决什么问题、适合谁、为什么值得看、风险在哪里。
- 如果信息源不可用，要显示“暂不可用/待接入”，不要伪造数据。
- 对外展示尽量使用标题、来源、链接和自生成摘要，不搬运全文。

## Implementation Notes

- 当前项目基于 Astro。
- 主要趋势样式集中在 `src/components/trends/TrendShell.astro`。
- 榜单组件在 `src/components/trends/RankBoard.astro`。
- 趋势详情页在 `src/pages/trends/[slug].astro`。
- 项目详情页在 `src/pages/projects/[slug].astro`。
